# Learning Memory

このファイルは、PHP-Studyでの学習状況をCodexに引き継ぐための中心ファイルです。

Codexはこのファイルを読むことで、現在の進捗、つまずき、理解できていること、まだ曖昧なことを把握します。

## 運用ルール

- このファイルは学習ログとして随時更新する。
- 既存の教材ファイル、課題ファイル、サンプルファイルは変更しない。
- 記録するときは、事実と推測を分ける。
- 「できない」ではなく「次に確認すること」として書く。
- 答えそのものではなく、学習状態と次のヒント方針を残す。
- システム構築以外の学習相談は、原則として `learning-memory/interaction-log.md` に記録する。
- 評価を行った場合は、原則として `learning-memory/evaluation-log.md` に記録する。

## わかっていること

- PHPファイルは `php ファイル名.php` で実行できる。
- 変数に数値を入れて、その値によって表示を変える流れは理解し始めている。
- `if`, `else`, `elseif` の基本的な形は使えている。
- `echo` で文字列を表示できる。
- 複数の条件を上から順番に判定する考え方に触れている。
- `for` 文の開始値、継続条件、増加式を問題文に対応させる考え方に触れている。
- 二重ループで外側と内側の値を組み合わせ、連番を作る考え方に進めている。

## つまずいていたこと

- 条件文の「日本語の条件」を、正確な比較式に置き換えるところ。
- `10より大きい` と `10以上` のような境界条件の違い。
- 偶数・奇数判定で、割り算の結果を見るのか、割った余りを見るのかの区別。
- たまたま現在の値で正しい出力に見えても、別の値で試すと正しくない可能性があること。
- ループの反復式で、計算するだけではなく変数の値を更新する必要があること。
- 合計用変数を使う前に初期化する必要があること。

## まだ曖昧そうなこと

- 比較演算子の使い分け: `>`, `>=`, `<`, `<=`, `==`
- 余りを調べる演算子の使い方。
- 条件式を作る前に、テストする値を複数考える習慣。
- 「問題文どおりの条件」と「自分が書いた条件」が一致しているか確認する方法。
- `for` の3つ目に書く「次の値への更新」と、ループ内で行う計算の違い。
- ループ内の余計な出力を残さず、解法を1つに絞ること。

## 次にCodexが出すとよいヒント

- まず「何回繰り返すか」「毎回何を変えるか」「何を表示・加算するか」に分ける。

- 実行結果だけでなく、警告や余計な出力がないかも確認させる。

## 評価システム

- 評価基準: `docs/learning/evaluation.md`
- 評価ログ: `learning-memory/evaluation-log.md`
- 評価は「最低要件」と「エンジニア視点」を分けて行う。
- 最低要件では、構文エラーがないこと、問題文どおりに動くこと、指定された構文を使うことを確認する。
- エンジニア視点では、シンプルさ、読みやすさ、余計な処理の少なさ、境界条件への意識を確認する。
- 評価しても、完成コードはすぐに出さない。
- 評価コメントでは、次に直すべき1点を優先する。

## メモリー記録システム

- 通常ログ: `learning-memory/interaction-log.md`
- 評価ログ: `learning-memory/evaluation-log.md`
- 更新ルール: `learning-memory/update-rules.md`
- 対象: システム構築以外の質問、ヒント依頼、エラー相談、評価依頼、答え合わせ相談。
- 除外: `AGENTS.md` や学習メモリー構造そのものを作る・変える依頼。
- 記録内容: 対象ファイル、質問の要約、つまずき、理解できていること、次回の支援方針。
- 禁止: 完成コード、最終解答、丸写しできる修正内容の記録。
