# Evaluation Log

このファイルは、課題コードを評価した記録を残す場所です。

評価基準そのものは `docs/learning/evaluation.md` に置き、このファイルには現在の状態や次に見るべき点を記録します。

## 現在の評価方針

- 最低要件とエンジニア視点を分けて評価する。
- 答えの完成コードは書かず、次に確認する1点を残す。
- 構文エラーがある場合は、まず実行可能性を最優先にする。
- 一見正しい出力でも、別の値で試したときに要件を満たすかを見る。
- 評価を行った場合は、原則としてこのファイルに記録する。
- 完成コードや最終解答そのものは記録しない。


## 次回以降の評価時に見ること

1. まず `php -l 対象ファイル` で実行可能性を見る。
2. 次に問題文と現在の条件式・処理を照合する。
3. その後、最低要件を満たしているか判定する。
4. 最後に、シンプルで読みやすいかをエンジニア視点で見る。
5. 評価コメントは、改善点を1つに絞って返す。
