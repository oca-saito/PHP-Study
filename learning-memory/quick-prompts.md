# Quick Prompts

毎回、次のように短く依頼できます。

通常の学習相談、ヒント依頼、評価依頼は、原則としてメモリーに記録されます。
システム構築の相談だけは通常ログから除外します。

## 学習状態を読ませてから相談する

```text
LEARNING_MEMORY.md を読んで、今の自分の進捗を前提に q3 のヒントだけください。
```

## 学習ログを更新する

```text
今の会話をもとに LEARNING_MEMORY.md を更新してください。既存の課題ファイルは変更しないでください。
```

## 理解できていることを整理する

```text
LEARNING_MEMORY.md を読んで、今わかっていることと曖昧なことを整理してください。
```

## 次にやる課題を決める

```text
LEARNING_MEMORY.md を読んで、次に取り組むべき課題を答えなしで提案してください。
```

## つまずきだけ記録する

```text
今つまずいた内容だけ LEARNING_MEMORY.md に記録してください。
```

## 課題を評価してほしい

```text
docs/learning/evaluation.md を基準に、今のファイルを答えなしで評価してください。
```

## 最低要件だけ見てほしい

```text
今のコードが最低要件を満たしているかだけ見てください。完成コードは不要です。
```

## エンジニア視点で見てほしい

```text
今のコードをエンジニア視点で、シンプルで読みやすいか評価してください。修正案は考え方だけでお願いします。
```

## 評価ログを更新する

```text
今の評価結果を learning-memory/evaluation-log.md に記録してください。
```

## 通常ログを確認する

```text
learning-memory/interaction-log.md を読んで、前回の続きからヒントをください。
```
